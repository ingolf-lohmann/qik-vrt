# Evolution Policy

Self-evolution means controlled improvement of repository quality, not uncontrolled self-modification.

Permitted autonomous outputs:

- defect reports,
- missing-requirement reports,
- proposed patches,
- test expansion proposals,
- documentation improvement proposals,
- compatibility reports,
- regression warnings,
- rollback plans.

Forbidden autonomous outputs:

- unsupervised normative changes,
- silent code changes,
- hidden deployment,
- remote modification without authorization,
- deletion of nonregression anchors,
- removal of boundary gates,
- weakening of privacy, proportionality or emergency safeguards.

Evolution gate:

1. identify difference,
2. classify requirement impact,
3. propose change,
4. run tests,
5. compare against nonregression anchor,
6. produce audit log,
7. require governance approval for adoption.

Continuous cognitive improvement is implemented as continuous evaluation of evidence classes, failure patterns, test gaps and traceability gaps. It is not implemented as uncontrolled autonomous agency.
