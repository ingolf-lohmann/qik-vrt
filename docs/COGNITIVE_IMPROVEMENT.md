# Cognitive Improvement Layer

The repository may improve its own operating knowledge only by explicit, testable and reversible mechanisms.

Cognitive improvement means:

- better classification of assertions,
- better detection of traceability gaps,
- better mapping of articles to implementation gates,
- better separation of fact, hypothesis, risk, interpretation and speculation,
- better regression detection,
- better human-readable explanation,
- better multicast routing to zuständige Knoten.

It does not mean:

- claiming consciousness,
- bypassing human rights,
- producing secret objectives,
- self-authorizing beyond the manifest,
- replacing legal, medical or institutional responsibility.

Improvement metric set:

- TRACE_COVERAGE,
- REQUIREMENT_COVERAGE,
- TEST_COVERAGE,
- BOUNDARY_STRENGTH,
- FALSE_PASS_RESISTANCE,
- HUMAN_READABILITY,
- REVERSIBILITY,
- PRIVACY_PRESERVATION.

A cognitive improvement is valid only if it increases at least one metric without reducing any boundary or nonregression gate.
