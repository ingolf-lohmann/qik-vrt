# Windows Final Acceptance Aggregation

The Windows runner records every attempt, including failed compiler candidates. This is intentional traceability.

A failed intermediate compiler attempt is not a final acceptance failure if a later authorized runtime bootstrap produces a working compiler and the verifier plus all required selftests pass.

Final acceptance is therefore computed from required final gates:
- repository structure
- SHA256SUMS
- JSON/JSONL parse
- live GitHub seed fetch
- dependency bootstrap availability
- successful C build
- repository verifier
- root layout validation
- live seed manifest validation
- static evidence validation
- every required C selftest
- result persistence

Recovered attempt-level records such as `C_BUILD_ATTEMPT BLOCK` remain in the audit trail, but do not poison final acceptance when superseded by `C_BUILD PASS`.
