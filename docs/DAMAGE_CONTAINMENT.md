# Damage containment

Damage containment is the safe response to a failed or missing repository selftest.

## Rules

- A selftest must be requested before containment.
- Failed selftest creates quarantine-required status.
- Missing heartbeat creates node-lost pending review status.
- An authorized multicast notice informs the responsible peer group.
- The containment ledger records node id, reason, observed time and trace state.
- There is no remote mutation.
- There is no destructive cleanup of another repository.
- Remediation requires owner-local repair or explicit governance authorization.
- Every containment action requires root-cause review.

## Purpose

The layer helps the repository network contain, encircle and remove harm from the network by evidence and isolation rather than by uncontrolled autonomous repair.
