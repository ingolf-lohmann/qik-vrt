# V1.9 correction: GitHub release-by-tag 404 handling

V1.9 incorrectly treated GitHub API `GET /repos/{owner}/{repo}/releases/tags/{tag}` returning `404 Not Found` as a fatal BLOCK.

That 404 is the expected state when no release exists yet. V1.9 changes the logic:

- repository lookup errors remain BLOCK;
- release-by-tag 404 is treated as `release_absent`;
- if a release already exists, the script blocks to avoid duplicate publication;
- if no release exists, the script continues to push branch/tag, create the GitHub release, and upload the PDF and release bundle as assets.

Status: `V1_9_RELEASE_404_EXPECTED_ABSENCE_FIXED`.
