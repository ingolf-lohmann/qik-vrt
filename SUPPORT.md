# Support

QIK-VRT is research software. Community support is best-effort and does not
create a service-level, professional-advice, or production-certification
obligation.

- Start with [README.md](README.md), [STATUS.md](STATUS.md), and
  [docs/CURRENT_AUTHORITY.md](docs/CURRENT_AUTHORITY.md).
- Run `make test` for the complete local verification gate.
- Use a bug-report issue for a reproducible software defect.
- Use a documentation issue for unclear or inconsistent documentation.
- Use a research-claim review issue when a claim, premise, derivation, or
  evidence boundary needs examination.
- Follow [SECURITY.md](SECURITY.md) for vulnerabilities.

Commercial licensing or contracted support requires a separate written
agreement with the rights holder; repository access or an issue response does
not grant one.
