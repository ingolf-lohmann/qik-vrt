# Einzigartige Merkmale von QIK-VRT Trust Mesh

## 1. Repository-native Evidenz

QIK-VRT schreibt Nachweise direkt in Repositories. Das macht Evidence versionierbar, öffentlich prüfbar, maschinenlesbar und revisionsfähig.

## 2. Seed/Node-Mesh

Ein Seed führt Registry, Index, Status, Revalidation und Audit. Nodes melden Health, Renewal und Seed-Acceptance. So entsteht ein Vertrauensnetzwerk, kein isolierter Prüfbericht.

## 3. Offene Multi-Node-Architektur

Neue Nodes werden nicht vorher gezählt. Sie werden über eine offene Queue und autorisierte Registry-Zeilen aufgenommen. Das Mesh bleibt wachstumsfähig.

## 4. Vollautomatischer Audit-Ablauf

Der aktuelle Ablauf reicht von Node-Renewal bis Release-Freeze:

```text
Node Renewal
Node Health
Seed Registry Acceptance
Node Seed Status Ack
Seed Mesh Maintenance
Seed Node Revalidation
Seed Audit Export
Seed Dashboard Publish
Node Self Audit
Release Freeze
```

## 5. Rechtlich anschlussfähige Ergebnisakten

QIK-VRT erzeugt strukturierte Nachweise, die für Audits, Due Diligence, Compliance, Management-Entscheidungen und juristische Bewertung verwertbar sind.

## 6. Boundary by Design

- kein globales Scanning
- keine Selbstverbreitung
- keine fremde Repository-Mutation durch Workflows
- kein Token im Repository
- Product Owner autorisiert
- jedes Repository schreibt nur in sich selbst

## 7. Künstliche Kognition als Ordnungsmodell

QIK-VRT ist nicht nur Automatisierung. Es operationalisiert künstliche Kognition als strukturierte Verarbeitung von Unterschieden, Informationen, Zuständen, Grenzen und Wirkungen.
