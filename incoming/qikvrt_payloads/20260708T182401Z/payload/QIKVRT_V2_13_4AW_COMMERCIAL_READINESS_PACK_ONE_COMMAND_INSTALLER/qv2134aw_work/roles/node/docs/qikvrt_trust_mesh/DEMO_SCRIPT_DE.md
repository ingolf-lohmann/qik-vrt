# Demo-Skript — 30 Minuten

## Minute 0-3: Problem

„Sie haben Repositories, Releases, Workflows, KI-Komponenten und Dokumentation. Aber können Sie morgen beweisen, welche Version aktiv war, wer sie autorisiert hat, welche Node welchem Seed vertraut, ob der Heartbeat aktiv war und welcher Audit-Run den Zustand bestätigt?“

## Minute 3-8: QIK-VRT Trust Mesh zeigen

- Seed-Repository öffnen
- NODEMESH_INDEX.json zeigen
- NODEMESH_STATUS.json zeigen
- NODEMESH_REVALIDATION.json zeigen
- Dashboard zeigen
- Audit-Report zeigen

## Minute 8-15: Node zeigen

- NODE_HEALTH.json
- NODE_REGISTRATION_RENEWAL.json
- SEED_ACCEPTANCE_STATUS.json
- Node Self Audit

## Minute 15-20: Compliance-Landkarte

- AI Act: technische Dokumentation, Logging, Transparenz, Human Oversight, Cybersecurity
- NIS2/CRA: Sicherheits- und Lieferkettennachweise
- QIK-VRT: Evidence statt Behauptung

## Minute 20-25: Business Case

- weniger manuelle Audit-Vorbereitung
- schnellerer Kunden-/Partnernachweis
- weniger Release-Unsicherheit
- bessere Product-Owner-Verantwortung

## Minute 25-30: Angebot

„Wir starten mit einem Repository Trust Audit. Danach entscheiden Sie, ob wir ein Seed/Node-Mesh aufbauen.“
