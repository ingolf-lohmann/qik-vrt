# Sales Call Playbook

## Einstieg

„Wir helfen Organisationen, Repository-Zustände, KI-Nachweise und Audit-Evidence automatisch und nachvollziehbar zu erzeugen.“

## Diagnosefragen

1. Wie weisen Sie heute nach, welche Repository-Version produktiv relevant war?
2. Gibt es eine maschinenlesbare Evidence-Kette?
3. Wie dokumentieren Sie KI-/Automatisierungsfreigaben?
4. Wer kann einen Audit-Run reproduzieren?
5. Wie erkennen Sie veraltete oder nicht mehr valide Nodes?
6. Gibt es ein Dashboard für Compliance und Technik zugleich?

## Wertversprechen

„QIK-VRT macht aus GitHub-Repositories prüfbare Vertrauensknoten.“

## Abschlussfrage

„Sollen wir mit einem begrenzten Repository Trust Audit beginnen?“
