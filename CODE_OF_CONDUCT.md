# Code of conduct

QIK-VRT welcomes careful technical review, reproducibility reports, security
findings, documentation feedback, and clearly delimited research criticism.
Participation in this repository requires respectful, evidence-oriented
conduct.

## Expected conduct

- Address claims, code, evidence, and methods rather than attacking people.
- Distinguish observation, inference, hypothesis, and established result.
- State conflicts of interest and relevant uncertainty.
- Protect personal data, credentials, and non-public vulnerability details.
- Respect the repository's contribution and mixed-license boundaries.
- Accept correction when a stronger reproducible result is presented.

## Unacceptable conduct

Harassment, threats, discriminatory language, doxxing, impersonation,
deliberate misrepresentation of evidence, publication of secrets, and attempts
to pressure participants into accepting unverified claims are not permitted.

## Reporting and enforcement

For ordinary conduct concerns, contact the repository owner through the
owner's GitHub profile without publishing sensitive details. Security issues
must follow [SECURITY.md](SECURITY.md). The repository owner may edit, hide, or
remove material; limit participation; or block accounts when necessary to
protect people, evidence, or repository integrity. Enforcement decisions
should identify the rule applied and preserve an appropriate audit trail.

This code applies to repository issues, pull requests, discussions, releases,
and project-controlled communication spaces.
